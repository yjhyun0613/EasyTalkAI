팀원이 알게 하려면 “내가 만든 기능”을 디자인 설명이 아니라 연동 명세서처럼 정리해서 넘겨야 합니다.
팀원이 프론트 담당이면, 네가 설명해야 할 핵심은 이겁니다.
나는 Android Native 쪽에서
공유받기 / 파일 처리 / 개인정보 동의 / WebView 연결 / Android 기능 브릿지를 만들었다.

프론트는 WebView 안에서 화면을 만들면 되고,
Android가 전달해주는 데이터와 호출 가능한 함수만 맞춰 쓰면 된다.아래 내용을 그대로 팀원에게 보내면 됩니다.

팀원 전달용 요약
현재 앱 구조는 Android Native + WebView 하이브리드 구조입니다.

내가 담당한 부분은 Android Native 기능 레이어입니다.

내가 구현한 것:
1. 앱 첫 실행 온보딩 화면
2. 앱 정보/개인정보 동의 체크
3. 동의하지 않으면 앱 시작 불가
4. Android 공유 Intent 수신
   - 문자/카카오톡 텍스트 공유
   - 이미지 공유
   - PDF 공유
   - 다중 파일 공유
5. 공유받은 파일을 앱 캐시에 저장
6. 앱 재시작 후에도 공유 파일 목록 복구
7. 파일 크기/형식 제한 처리
8. WebView로 공유 텍스트/파일 목록 전달
9. WebView에서 Android 기능 호출 가능하게 브릿지 구현
10. 결과 복사 / 가족에게 공유 / 캘린더 저장 기능 연결
11. Debug/Release 분리
12. Release에 개발용 bridge_test.html이 들어가지 않도록 관리

프론트 담당자는 WebView 안에서 화면을 만들면 됩니다.
Android 쪽에서 아래 데이터와 함수를 제공합니다.

Android가 프론트로 보내는 데이터
프론트에서는 아래 방식 중 하나로 데이터를 받으면 됩니다.
1. 함수 방식

window.EasyTalkAI.receiveSharedPayload = function(payload) {
  console.log(payload.text);
  console.log(payload.files);
};Android가 전달하는 payload 구조:

{
  text: "사용자가 공유하거나 입력한 안내문 텍스트",
  files: [
    {
      fileId: "cached_file_id",
      name: "notice.pdf",
      mimeType: "application/pdf",
      sizeBytes: 123456,
      sizeText: "120.6KB"
    }
  ]
}2. 이벤트 방식

window.addEventListener("easyTalkAISharedPayload", function(event) {
  const payload = event.detail;

  console.log(payload.text);
  console.log(payload.files);
});3. localStorage 방식

const text = localStorage.getItem("easyTalkAI.sharedText");
const files = JSON.parse(localStorage.getItem("easyTalkAI.sharedFiles") || "[]");

EasyTalkAINative.copyText("정리 결과", resultText);결과 공유

EasyTalkAINative.shareText("쉽게말해 AI 정리 결과", resultText);캘린더 저장

프론트가 맡으면 되는 부분 (대화형 레이아웃 구현 완료)
프론트 담당자가 구현 완료한 사항:
1. ChatGPT/Gemini 스타일의 단일 대화창 화면 구현
2. 대화 말풍선 내에 요약 리포트 카드(한 줄 요약, 할 일 체크리스트, 중요 일정, 준비물) 인라인 렌더링
3. 요약 리포트 카드 바로 하단에 🔊 읽어주기, 📅 달력 등록, 💬 보호자 공유 버튼 연동
4. 요약 리포트 생성 후 문서 맥락에 맞춘 실시간 꼬리질문 대화 기능 구현 (Gemini API 및 시뮬레이션 연동)
5. 이미지/PDF 파일 첨부 시 대화창 상에서 즉시 로딩 애니메이션 노출 및 자동 분석 트리거 적용
6. 안드로이드로부터 공유 Payload 수신 시 대화방 흐름에 맞춰 자동 요약 실행되도록 브릿지 결합

내가 만든 Android 쪽 주요 파일
app/src/main/java/com/easyai/app/MainActivity.kt
→ Android Native 기능 대부분
→ 공유 Intent 수신, 파일 캐시, WebView 브릿지, 온보딩, 동의, 입력 확인 화면

app/src/main/res/layout/activity_main.xml
→ Native 화면 레이아웃
→ 온보딩, 개인정보 동의, 공유 내용 확인 화면

app/src/main/res/drawable/
→ 온보딩/입력 확인 화면 배경, 카드, 버튼 스타일

app/src/debug/assets/bridge_test.html
→ 개발/테스트용 WebView 화면
→ Android ↔ WebView 연결 확인용

app/src/release/assets/production_placeholder.html
→ Release에서 사용할 실제 프론트 화면 자리

Debug / Release 구분
Debug:
app/src/debug/assets/bridge_test.html
→ 개발용 테스트 화면

Release:
app/src/release/assets/production_placeholder.html
→ 실제 프론트 결과물 넣을 위치프론트 팀원이 만든 결과 HTML/JS/CSS를 Release에 넣을 때는 보통 여기를 교체하면 됩니다.
