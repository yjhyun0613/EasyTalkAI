# 쉽게말해 AI - 안드로이드 앱 ↔ 웹뷰 프론트엔드 연동 계획서

이 문서는 모바일 웹앱(하이브리드 앱) 개발을 성공적으로 완수하기 위해 안드로이드(Android Native) 개발자와 웹프론트엔드(WebView) 개발자 간의 기능 연동 및 테스트 절차를 정리한 연동 계획서입니다.

---

## 1. 개요 및 현재 개발 현황

- **앱 아키텍처**: Android Native Shell + WebView 기반 하이브리드 앱 구조
- **웹프론트엔드 구축 현황**:
  - 공식 랜딩 페이지(`index.html`) 및 예시 데모 요약 페이지(`demo.html`) 퍼블리싱 완료.
  - 스마트폰 홈 화면 추가 및 오프라인 고속 실행을 위한 **PWA(프로그레시브 웹앱) 표준 구성 완료** (`manifest.json`, `sw.js`).
  - 모바일 세로 뷰포트에 맞춘 **반응형 최적화 및 터치 영역(Touch Target, 최소 48px) 설계 적용 완료**.

---

## 2. 웹프론트엔드 측 안드로이드 브릿지 사전 반영 완료 사항

안드로이드 개발팀이 작성한 `핸드폰연동.md` 명세에 맞추어, **프론트엔드 웹 코드(`app.js` 및 `demo.html`) 내에 네이티브 브릿지 분기 처리를 100% 선반영 완료**했습니다.

### ① 네이티브 기능 연동 분기 (자동 감지)
웹 내부에서 글로벌 객체 `EasyTalkAINative` 존재 여부를 실시간 감지하여 다음과 같이 네이티브 API를 호출합니다.

- **결과 복사**: 
  - `btnShareCopy` 클릭 시, `EasyTalkAINative.copyText("쉽게말해 AI 요약 결과", text)`를 자동으로 호출합니다. (앱이 아닐 시 일반 웹 복사 기능으로 작동)
- **결과 공유**: 
  - `btnShareKakao` 클릭 시, `EasyTalkAINative.shareText("쉽게말해 AI 정리 결과", text)`를 호출하여 안드로이드 네이티브 공유 다이얼로그를 기동합니다.
- **캘린더 일정 저장**:
  - 네이티브 앱 감지 시 웹 범용 일정 다운로드 버튼을 숨기고 **"📱 스마트폰 기본 캘린더에 추가"** 전용 버튼으로 화면이 자동 전환됩니다.
  - 클릭 시 `EasyTalkAINative.addCalendarEvent(title, dtStartMs, dtEndMs, description, location)`를 밀리초 문자열 타임스탬프 형식으로 파라미터를 파싱하여 안전하게 전달합니다.

### ② 외부 앱 공유 수신 처리 (공유 Intent 수신)
안드로이드 네이티브가 외부(문자메시지, 카카오톡, PDF 파일 등)로부터 수신한 페이로드를 웹뷰로 넘겨주면, 아래 세 가지 수신 채널을 감지하여 입력 창에 텍스트를 자동으로 적재하고 분석 준비 모드로 자동 진입합니다.

1. **글로벌 함수 채널**: `window.EasyTalkAI.receiveSharedPayload(payload)`
2. **이벤트 리스너 채널**: `easyTalkAISharedPayload` 커스텀 이벤트 감지
3. **로컬 스토리지 채널**: `localStorage.getItem("easyTalkAI.sharedText")` 감지

## 3. 향후 협업 및 연동 테스트 단계 (Step-by-Step)

두 개발자가 다른 지역에서 원격으로 연동 테스트를 진행하기 위한 단계별 절차입니다.

### **[Step 1] 웹 리소스 소스코드 전달**
- 프론트엔드 개발자가 작성한 웹 리소스 패키지(HTML, CSS, JS, 이미지 자산) 전체를 안드로이드 개발자에게 전송합니다.
  - 주요 폴더 구조:
    - `./index.html` (공식 랜딩 페이지)
    - `./demo.html` (웹앱 시연 페이지)
    - `./style.css` (반응형/고대비 스타일)
    - `./app.js` (네이티브 브릿지 내장 로직)
    - `./mockData.js` (시연용 Mock JSON)
    - `./manifest.json` & `./sw.js` (PWA 설정)
    - `./app_icon.png` (512x512 고해상도 앱 로고)

### **[Step 2] 안드로이드 프로젝트 내 파일 배치**
- 안드로이드 개발자는 전달받은 웹 소스 파일을 프로젝트 내 에셋 경로에 임베딩합니다.
  - **Debug 빌드**: `app/src/debug/assets/` 경로에 배치 (개발용 테스트)
  - **Release 빌드**: `app/src/release/assets/` 경로에 배포용 파일 배치 (실제 배포)

### **[Step 3] 브릿지 연동 디버깅**
- 앱을 실행하여 웹뷰가 로드되었을 때 `EasyTalkAINative` 객체가 정상적으로 인젝션(주입)되는지 확인합니다.
- 웹뷰 콘솔 로그 디버깅을 활성화하여 브릿지 호출 시 오류가 발생하지 않는지 체크합니다.

### **[Step 4] 엔드투엔드(E2E) 공유 시나리오 테스트**
- 스마트폰 문자나 알림톡에서 임의의 복잡한 텍스트를 드래그하여 **'공유하기 -> 쉽게말해 AI 앱 선택'**을 실행합니다.
- 네이티브가 Intent를 캐치하여 웹뷰를 로드하고, 웹뷰 입력창에 해당 텍스트가 자동으로 로드되는지 확인합니다.
- 요약 완료 후 복사, 공유, 캘린더 등록을 눌러 안드로이드 시스템 알림 및 캘린더 앱에 일정이 동기화되는지 최종 확인합니다.
